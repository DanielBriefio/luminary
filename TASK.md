## Step 2 ‚Äî Sync CLAUDE.md with remaining snapshot findings

### New tables to add to Database Schema section

**`group_follows`**
```
id, group_id, user_id, created_at
UNIQUE(group_id, user_id)
```
Note: used for users following a group without joining it. Separate
from the `follows` table (which handles user + paper + group follows
via `target_type`).

**`group_invites`**
```
id, group_id, token, max_uses (default 10), expires_at (7-day),
created_by, created_at
```
Note: token-based group invite links. RLS: insert/update/delete
restricted to group admins. Distinct from platform-wide `invite_codes`
(which gates platform signup ‚Äî different table, different purpose).

### Missing columns to add to existing table docs

**`group_members`**: add `last_read_at (timestamptz)` ‚Äî supports
unread-count tracking per group member. Same pattern as
`project_members.last_read_at`.

**`groups`**: add `slug (TEXT UNIQUE)` ‚Äî auto-generated by
`generate_group_slug` trigger. Used by `/g/:slug` public group route
(`PublicGroupProfileScreen`). Confirm trigger name by running:
```sql
select trigger_name, event_manipulation, action_statement
from information_schema.triggers
where event_object_table = 'groups'
  and trigger_schema = 'public';
```
Document the trigger name accurately.

**`posts`**: add `group_id, group_name, is_deep_dive` ‚Äî revealed by
`posts_with_meta` view. `group_id`/`group_name` stamp the source group
on a public repost; `is_deep_dive` flags the deep-dive post variant.

**`project_posts`**: add `content_iv (TEXT), content_encrypted (BOOL),
tier1 (TEXT), tier2 (TEXT[]), edited_at (TIMESTAMPTZ)` ‚Äî encryption
columns match the Phase 6 placeholder pattern on `posts` and
`group_posts`; `tier1`/`tier2` support discipline tagging; `edited_at`
matches other post tables.

### Views to update

**`groups_with_stats`** (new ‚Äî not previously documented):
Aggregates per group: `member_count, admin_count, alumni_count,
active_member_count, publication_count`. Used by
`PublicGroupProfileScreen` and group discovery.

**`posts_with_meta`** (update existing entry): add
`repost_count, user_liked, user_reposted, author_work_mode,
author_slug` to the documented column list.

**`project_posts_with_meta`** (update existing entry): add
`project_name, project_icon, project_color, project_group_id`
to the documented column list.
